E-mail cheats

This is a quick guide to show you how to create some elements in your e-newsletters.

Tables

Let's start making some tables!

There are a few elements necessary to create a table. In order:

  • We start with table which defines the table
  • This is followed by tbody
  • Each row needs to be defined with tr
  • ...and each cell with td

Great — now it's starting to look like a real table. We also have a few things we need to add to table to make it look a bit nicer, namely cellpadding="0", cellspacing="0", margin="0" and border="0". We also might want to make our cells the same size, which we can do by putting width="50%" in each td. You can also use more specific definitions, such as width="100". Default measurement will be in pixels.

Pro tip: You don't have to define the width in each row; subsequent rows will inherit the width defined in the first row.

See the Pen taEFu by Audrey Saccone (@audreysaccone) on CodePen.

Let's add a little bit more styling! Remember: if you need more information on CSS styling and what does what, you can check out this cheat sheet.

  • Enclose all CSS styling with style=""
  • Change your font in the cell to bold by using font-weight:bold;
  • Change your font in the cell to italics by using font-style:italic;
  • Define your font and font size when writing tables in e-mails with font-family:Arial; and font-size:12px; (or whatever font family and size you are using!). You only have to do this in the table element and the rest of the table will inherit the property.
  • We can also add some padding, which adds white space (or whatever your background color is) to the inside of your cell.
  • A full string of CSS styles may look like style="font-weight:bold;font-family:Arial;font-size:12px;padding:10px;"

Any CSS in a td will override the CSS in the tr or table elements, which you'll see in the next example.

A few more things before you start making your own table. We also might want to add a table with cells that cross over rows and columns. This is a bit more advanced, but not too hard. To make a cell two rows high, we have to add rowspan="2". To make a cell two columns wide, we have to add colspan="2". Make sense?

Both of these elements can be combined to make a double tall + wide cell (or more; your value doesn't always have to be 2). But remember: each row and column have to add up across the whole table, otherwise your table will become confused. Each cell in the example below has a unique value in the cell so that you can match the Result and the HTML easily.

See the Pen LdBfb by Audrey Saccone (@audreysaccone) on CodePen.

Note: All of the tables above have a border. I usually dont' use borders in tables in e-mails; they're pretty unnecessary. But, if you need them, you now know have to do them! Both borders and padding can be defined as a whole (by using just style="border:1px solid black" or style="padding:10px;") or by individual sides. You do this by appending the side to border or padding as such: style="border-left:2px solid red;border-right:1px solid green;padding:10px;padding-right:15px;". A side-specific property willl override an all-sides property, so in the previous example all sides have padding of 10px except for the right, which has padding of 15px.

Now go forth and make some tables!
